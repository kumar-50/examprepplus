# 05 Landing Page Skeleton

## Objective

Introduce initial marketing surface: hero section + CTA with responsive layout.

## Steps

1. Create route `/` with server component shell.
2. Add hero: headline, subtext, primary CTA (Get Started) linking signup.
3. Set up color tokens & spacing scale.
4. Ensure mobile-first layout (stacked) then wide variant.
5. Add basic meta tags (title, description placeholder).

## Acceptance Criteria

- Page renders without layout shift (CLS minimal).
- CTA navigates to auth page.

## Best Practices

- Server components for static marketing sections.
- Limit blocking fonts; use system stack initially.
